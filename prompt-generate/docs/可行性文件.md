# Prompt Generator 可行性文件

> 建立日期：2026-05-25  
> 專案代號：`prompt-generate`  
> 部署目標：GitHub Pages（純前端、靜態網站）

---

## 1. 專案目標

打造一個**模組化的 Prompt 產生器**，讓使用者可以像組合「物件導向元件」一樣，自由抽換以下面向：

- 角色（Character）
- 服裝（Outfit）
- 身材（Body）
- 姿勢（Pose）
- 場景（Scene）
- 光影 / 畫風（Lighting / Style）
- 鏡頭 / 構圖（Camera）
- 負面 Prompt（Negative）
- 母版 / 比例（Template，如 16x9、9x20）

最終於同一個 HTML 頁面中：
1. 透過下拉式選單、卡片、Tab 等 UI 進行選取與微調。
2. 即時預覽組合後的完整 Prompt。
3. 一鍵複製到剪貼簿，貼到 AI 繪圖工具中使用。
4. 所有自定義模組與「成品 Prompt」可儲存於瀏覽器 `localStorage`，不需後端。
5. 可匯出 / 匯入 JSON，方便在不同裝置或備份到 Git 倉庫。

## 2. 核心設計理念：把 Prompt 當成物件

借鑑物件導向（OOP）的「組合優於繼承」概念：

```text
FinalPrompt = Template
            + Character
            + Outfit
            + Body
            + Pose
            + Scene
            + Lighting
            + Camera
            + ExtraTags
            - Negative
```

每個模組為一個獨立 JSON 物件，擁有相同的基底欄位（id、name、tags、weight、note…），可被自由替換、複製、修改。

### 2.1 模組資料結構（草案）

```jsonc
{
  "id": "char_purple_hair_girl",
  "type": "character",          // character | outfit | body | pose | scene | lighting | camera | negative | template
  "name": "紫髮少女",
  "tags": [
    "1girl",
    "long purple hair",
    "violet eyes",
    "slim figure"
  ],
  "weight": 1.0,                 // 整體權重，輸出時可包成 (xxx:1.1)
  "negative": [],                // 此模組自帶的負面詞
  "note": "預設身高 160cm，氣質清冷",
  "thumbnail": "",               // 可選，base64 或圖片 URL
  "createdAt": "2026-05-25T10:00:00+08:00",
  "updatedAt": "2026-05-25T10:00:00+08:00"
}
```

### 2.2 組合輸出（FinalPrompt）

```jsonc
{
  "id": "final_2026-05-25_001",
  "title": "紫髮少女_摩天輪夜景_約會洋裝",
  "modules": {
    "template":  "tpl_16x9_anime",
    "character": "char_purple_hair_girl",
    "outfit":    "outfit_date_dress",
    "body":      "body_slim_petite",
    "pose":      "pose_sit_ferris_wheel",
    "scene":     "scene_ferris_wheel_night",
    "lighting":  "light_ferris_night",
    "camera":    "cam_medium_shot",
    "negative":  ["neg_common", "neg_hands"]
  },
  "extraTags": ["masterpiece", "best quality"],
  "renderedPrompt": "…組合後的最終字串…",
  "renderedNegative": "…負面 Prompt 字串…",
  "referenceImages": [],          // base64 或外部 URL，僅前端顯示
  "createdAt": "2026-05-25T10:00:00+08:00"
}
```

## 3. 功能需求

### 3.1 必要功能（MVP）

| #   | 功能              | 說明                                              |
| --- | ----------------- | ------------------------------------------------- |
| F1  | 模組 CRUD         | 新增 / 編輯 / 刪除 / 複製 各類模組                |
| F2  | 組合面板          | 以分頁或側欄選擇每個類別中的一個（或多個）模組    |
| F3  | 即時預覽          | 下方即時顯示組合後的 Prompt 與 Negative           |
| F4  | 一鍵複製          | 複製 Prompt / Negative / 全部                     |
| F5  | localStorage 儲存 | 模組、成品、設定全部存本機                        |
| F6  | 匯入 / 匯出 JSON  | 整包備份與還原                                    |
| F7  | 參考圖預覽        | 可貼上 / 拖曳圖片，僅顯示於頁面協助構圖（不上傳） |

### 3.2 進階功能（後續迭代）

- 模組權重微調滑桿（自動輸出 `(tag:1.2)` 格式）。
- 標籤搜尋 / 篩選 / 標籤雲。
- 多語系（中／英）標籤對照。
- 主題切換（深色 / 淺色）。
- 從現有 `08_成品Prompt/` 的 Markdown 一鍵匯入為模組。
- 與 `Prompt-Library/` 倉庫同步（透過匯出 JSON → 手動 commit）。
- PWA：可離線使用、加入主畫面。

## 4. 技術選型

| 面向 | 選擇                                                                      | 理由                                       |
| ---- | ------------------------------------------------------------------------- | ------------------------------------------ |
| 部署 | **GitHub Pages**                                                          | 免費、靜態、與 repo 整合                   |
| 框架 | **純 HTML + Vanilla JS + TypeScript（可選）** 或 **Vite + Vue 3 / React** | MVP 用 vanilla 即可；若元件變多再導入 Vite |
| 樣式 | **TailwindCSS（CDN）** 或自寫 CSS                                         | 快速、響應式                               |
| 儲存 | **localStorage**（主要）+ **IndexedDB**（圖片 base64 量大時）             | 純前端、零後端                             |
| 匯出 | JSON 檔案下載、剪貼簿 API                                                 | 標準 Web API                               |
| 圖示 | Lucide / Heroicons（SVG）                                                 | 輕量                                       |
| 打包 | MVP 可不打包；進階用 Vite                                                 | 簡單優先                                   |

> 建議第一版用「**單檔 index.html + 模組化 JS（ES Modules）**」，零建置即可直接放上 GitHub Pages。

## 7. 資料流程

```text
[使用者操作]
     │
     ▼
[UI 元件] ──觸發──► [Store（記憶體狀態）]
     │                       │
     │                       ├─► localStorage（持久化）
     │                       │
     │                       ▼
     └─────────◄──── [Composer：組合 Prompt 字串]
                              │
                              ▼
                       [預覽 / 複製 / 匯出]
```

- **單一資料來源（Single Source of Truth）**：所有模組與成品集中於 `Store`。
- **訂閱式更新**：UI 監聽 Store 變化，重新 render（可用簡易 pub/sub）。
- **persist 時機**：每次 Store 變更後 debounce 300ms 寫入 localStorage。

## 8. localStorage 規劃

| Key              | 內容                          | 備註             |
| ---------------- | ----------------------------- | ---------------- |
| `pg:version`     | schema 版本號                 | 用於未來資料遷移 |
| `pg:modules`     | 所有模組陣列                  | 主要資料         |
| `pg:finals`      | 成品 Prompt 陣列              | 歷史紀錄         |
| `pg:settings`    | 使用者設定（主題、預設母版…） |                  |
| `pg:lastCompose` | 上次組合狀態                  | 重新整理可還原   |

容量考量：
- localStorage 約 5MB 上限。
- 參考圖建議**不存 base64 進 localStorage**，改放記憶體或 IndexedDB；或僅存外部 URL。

## 9. 可行性評估

| 面向                | 評估                        | 風險 / 對策                   |
| ------------------- | --------------------------- | ----------------------------- |
| 技術可行性          | ⭐⭐⭐⭐⭐ 純前端即可達成        | 無後端、無金鑰問題            |
| 部署可行性          | ⭐⭐⭐⭐⭐ GitHub Pages 直接服務 | repo 設定 Pages 即可          |
| 資料保存            | ⭐⭐⭐⭐ localStorage 足夠      | 5MB 上限 → 提供匯出 JSON 備份 |
| 跨裝置同步          | ⭐⭐ 不原生支援               | 透過匯出 / 匯入或 commit JSON |
| 與現有 Library 整合 | ⭐⭐⭐⭐ 可解析既有 Markdown    | 提供 seed.json 與匯入工具     |
| 維護成本            | ⭐⭐⭐⭐ 低                     | 純前端、無依賴後端            |
| 擴充性              | ⭐⭐⭐⭐⭐ 模組化設計            | 新類別只需加 type 常數        |

**結論：可行性高，建議直接進入 MVP 開發。**
